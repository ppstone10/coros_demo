# S04-A 首页健康总览卡片

## 基本信息

- 所属实习生：S04
- 候选方案：A
- 项目方向：健康总览 / 数据聚合 / 卡片排序
- 推荐架构：MVVM 或 MVI

## 配套公共文档

请同时阅读 [公共培训要求](common-training-requirements.md)。公共文档包含项目来源、内测版本、Figma、KMP + Native UI 技术栈、安全边界、Protocol Buffers mock 数据结构、注册登录门禁、统一交付物、Codex 记录、评分和里程碑要求。

## 适用画像

适合产品感较好、能兼顾数据逻辑和展示体验的实习生。

## 项目目标

实现一个脱敏健康首页，聚合睡眠、今日运动、训练负荷、恢复状态等卡片，并根据数据状态输出展示优先级。

## 实现范围

- mock 数据源：daily summary、sleep summary、training load、recovery。
- 卡片类型：睡眠、今日运动、训练负荷、恢复状态、空态引导。
- 排序策略：有风险提醒优先、今日数据优先、无数据降级。
- UI model：每张卡片输出标题、摘要、状态、操作入口。
- mock 场景：正常、部分缺失、全空、异常值。

## 3 个月执行计划

| 周期 | 任务 |
| --- | --- |
| 第 1-2 周 | 完成注册 / 登录前置模块硬门禁，并结合本项目学习卡片 UI state、数据聚合和单测 |
| 第 3-4 周 | 设计卡片模型、排序规则、mock 场景数据 |
| 第 5-6 周 | 完成首页卡片可运行版本 |
| 第 7-8 周 | 补空态、异常态、优先级策略 |
| 第 9-10 周 | 抽离健康卡片聚合规则，补聚合测试 |
| 第 11-12 周 | 完成产品体验复盘和答辩 |

## 完成档位

- 基础要求：通过注册登录硬门禁，健康总览卡片可运行，业务 mock 数据源能验证未登录拦截。
- 标准要求：完成卡片聚合、排序、空态、错误态和本项目测试要求。
- 挑战要求：支持卡片配置、局部刷新、趋势摘要和可复用 dashboard state。

## 验收标准

- 正常、空数据、部分数据、异常数据均可展示。
- 卡片展示优先级可解释。
- 业务规则层输出稳定 UI model。
- UI 不直接拼多源数据聚合规则。

## 测试要求

- 至少 12 条单元测试。
- 必须覆盖全量数据、睡眠缺失、今日运动缺失、恢复状态异常、卡片排序。

## 观察点

- 是否能主动考虑空态和用户理解成本。
- 是否能把"展示优先级"写成可测规则。
- 是否能兼顾产品体验和工程分层。
