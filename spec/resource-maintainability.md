# 全模块资源可维护性 Spec

## 元数据

- Spec ID 前缀：`RES-MAINT`
- 状态：生产文案与视觉基础治理已完成，图片审计待后续批次
- 负责人：项目维护者
- 关联需求：`docs/resource-management.md`、两份平台资源管理参考指南、`spec/resource-localization.md`
- 最后更新：2026-07-17

## 目标

- 为 Android、iOS、HarmonyOS 和 `common` 建立可执行的全资源盘点，而不是依赖人工记忆资源分布。
- 明确设计源、平台运行资源、共享语义资源和平台专属资源的边界。
- 对三端共同使用的图片、图标、视频和 Lottie 建立语义清单与存在性校验。
- 量化用户可见硬编码文案和直接颜色值的存量债务，并使用只降不升的门禁支持分批迁移。
- 最终将生产页面用户可见文案迁入平台原生文字资源，将重复视觉值收敛为有限语义 Token。

## 非目标

- 应用内语言设置、切换与持久化由后续 `spec/app-language-switching.md` 独立治理；长英文视觉验收仍不属于本 Spec。
- 不删除、缩写或改变法律正文的中文语义；英文正文仅作为用户确认可接受的 Demo 翻译，不作为正式发布条款。
- 不把调试日志、路由名、序列化字段、资源文件名等非用户文案机械迁入本地化资源。
- 不为追求目录一致而移动 AppIcon、启动图或平台专属系统资源。
- 不在未完成引用核对前删除疑似重复图片。
- HarmonyOS `DebugStatePage` 是不进入正式 Demo 的内部诊断页，不要求多语言；其日志、测试标签和校验对照文字不计入生产文案债务。

## 边界与约束

- 运行时事实仍由各平台原生资源系统负责：Android `res`、iOS Asset/String Catalog、HarmonyOS `resources`。
- `login_register_resources/` 与 `health_dashboard_resources/` 仅作为设计与导入来源，不参与运行时读取。
- 资源清单记录语义名称和平台落点，不复制图片、视频或法律正文内容。
- 债务门禁采用“基线只降不升”：后续迁移应降低计数；确有新增用户文案时必须先进入平台资源，不能提高基线逃避检查。
- `common` 中领域模型、状态和数值保持平台无关；健康最终用户句子已拆成语义键与字符串参数。历史国家名称的“国家代码化 + 旧快照迁移”由 `spec/app-language-switching.md` 承接。

## 数据与状态

| 名称 | 结构 | 事实来源 | 约束 |
|------|------|----------|------|
| 共享图片清单 | 语义名数组 | 三端运行资源交集 + 页面引用 | 每个名称在三端均存在 |

## 行为规范

### `RES-MAINT-001`：全资源机器可读清单

- Given：项目包含三端图片、Raw 文件、文字资源和两组设计源。
- When：维护者读取资源清单或运行资源门禁。
- Then：能够得到共享图片、共享 Raw、平台专属资源原则和当前债务基线。
- 异常/边界：清单只保存名称与约束，不复制二进制内容。

### `RES-MAINT-002`：共享图片跨端存在性

- Given：某图片被声明为三端共享语义资源。
- When：运行资源维护门禁。
- Then：Android drawable/mipmap、iOS imageset、HarmonyOS media 中都能找到对应语义名。
- 异常/边界：AppIcon、LaunchLogo、启动器前景等平台专属资源不进入共享集合。

### `RES-MAINT-003`：共享 Raw 内容一致

- Given：`home.mp4` 与 `watch_status.json` 被三端共同使用。
- When：运行资源维护门禁。
- Then：三端同名文件存在且 SHA-256 一致，防止只替换某一端。
- 异常/边界：平台后续确需不同编码时，应创建显式平台变体规则，不能静默放宽检查。

### `RES-MAINT-004`：硬编码资源债务只降不升

- Given：生产源码仍有历史用户可见中文字符串和直接颜色值。
- When：运行资源维护门禁。
- Then：按 Android、iOS、HarmonyOS、common 分别计数，任一计数高于已审查基线时失败。
- 异常/边界：调试页与法律正文分别展示计数，后续独立迁移；不能通过提高基线让检查转绿。

### `RES-MAINT-005`：消除纯硬编码资源包装

- Given：项目目前存在 `AppText`、`AppColors` 等历史集中常量。
- When：逐批迁移生产页面。
- Then：文字使用平台原生资源，颜色/字体/间距使用有主题价值的有限语义 Token；不再新增只重复映射平台资源的包装。
- 异常/边界：`AppImages` 只有在承担跨状态选择、类型安全或统一行为时保留；纯名称映射后续改为原生直接引用。

### `RES-MAINT-006`：可执行资源维护门禁

- Given：资源、资源引用或生产 UI 发生变更。
- When：执行 `./tools/check-resource-maintainability.sh`。
- Then：清单格式、共享文件存在性、Raw 哈希和债务上限全部通过，并输出可审查摘要。
- 异常/边界：本命令补充而不替代三端构建、`check-resources.sh` 和人工视觉验收。

### `RES-MAINT-007`：法律正文作为结构化内容资源

- Given：隐私政策与服务条款正文当前在三端源码中重复维护。
- When：迁移法律正文。
- Then：正文进入三端原生本地化资源，中文内容完整保留，英文提供明确标记为 Demo 的对应翻译；页面代码只保留结构解析与展示逻辑，不再复制正文。
- 异常/边界：正文段落、标题和强调信息必须可恢复；正式发布前仍需法律审校，资源迁移不能被解释为条款有效性确认。

### `RES-MAINT-008`：健康内容使用结构化本地化契约

- Given：`common` 当前把健康标题、摘要、日期和原因拼成最终中文句子。
- When：共享健康用例生成 UI model。
- Then：共享层只输出稳定 `health_*` 语义键和字符串参数；三端展示边界使用平台原生资源格式化最终文案，领域数值、状态和排序规则继续由 `common` 决定。
- 异常/边界：`priorityReason` 同样使用稳定语义键；不得把 Android/iOS/HarmonyOS 资源类型引入 `commonMain`。

### `RES-MAINT-009`：调试资源与生产债务分离

- Given：HarmonyOS `DebugStatePage` 只用于组件、事件和 KNOI 对照调试。
- When：统计生产用户可见文案。
- Then：门禁显式排除该文件，并在清单中记录排除理由；正式生产页面不得借用此例外。
- 异常/边界：调试页仍参与 HarmonyOS Debug 构建，控制台日志和内部测试标签不进入共享资源清单。

### `RES-MAINT-010`：有限语义视觉 Token

- Given：三端页面存在重复颜色、字号和间距直接值。
- When：治理生产 UI。
- Then：建立颜色、Typography 和 Spacing 的有限语义 Token，并优先迁移健康、认证、账户及资料等重复值；保持现有视觉尺寸和颜色不变，不以本轮名义重新设计界面。
- 异常/边界：一次性布局计算值、Canvas 几何值、动画时长和系统要求尺寸可保留在组件内；Token 必须表达复用等级或语义用途，不能为每个数字创建一对一别名。
